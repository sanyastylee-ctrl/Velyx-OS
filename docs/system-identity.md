# Velyx System Identity

The user-facing product language for this stage is:

- Velyx OS
- Velyx OS Preview
- Velyx Installer
- Velyx First Boot
- Velyx Shell
- Velyx Update
- Velyx Recovery

## Identity rule

Linux-based substrate details may still exist underneath the preview, but they are implementation details.

User-facing flows should describe:

- Velyx install
- Velyx session
- Velyx shell
- Velyx update and recovery
- Velyx assistant and model setup

They should not frame Ubuntu or the base userspace as the primary product identity.

## Preview honesty

This is still a preview-stage operating environment:

- not a custom kernel
- not a consumer-grade installer on par with mature distros
- not a full package ecosystem
- not a complete hardware abstraction platform

But it is now intended to be perceived and operated as a standalone `Velyx OS Preview` experience.
